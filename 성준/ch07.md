# ✅ **정답 & 해설**

1️⃣ **C** — Compose 파일은 `version: "3.x"` 형식을 사용한다.

2️⃣ **B** — services는 애플리케이션을 구성하는 컨테이너 정의 섹션이다.

3️⃣ **B** — Compose 실행 명령은 `docker-compose up` 또는 `docker compose up`.

4️⃣ **C** — Compose는 프로젝트 단위 브리지 네트워크를 자동 생성하고 서비스명 DNS를 제공한다.

5️⃣ **B** — 환경 변수 지정 키워드는 `environment:`.

6️⃣ **B** — .env 파일은 구성 단순화 + 민감 정보 분리를 위해 사용한다.

7️⃣ **B** — Compose는 Swarm/Kubernetes 같은 오케스트레이션 기능이 부족하다.

8️⃣ **C** — 서비스 의존 관계 지정 키워드는 `depends_on`.

9️⃣ **B** — 백그라운드 실행은 `docker compose up -d`.

10️⃣ **A** — volumes는 services와 같은 상위 레벨에서 정의한다.
